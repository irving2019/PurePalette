Асинхронное WPF-приложение для независимой загрузки изображений из сети по URL. 
Разработано в рамках тестового задания на позицию C# / WPF разработчика.

## Технологии и стек
* **Платформа:** .NET 6.0 (C# 10)
* **Интерфейс:** WPF (Windows Presentation Foundation)
* **Архитектура:** Чистый MVVM (Model-View-ViewModel) без использования сторонних фреймворков.

## Ключевые особенности и архитектурные решения

* **Отсутствие дублирования кода (DRY):** В XAML-разметке не используется копипаст. Интерфейс карточек генерируется динамически с использованием `ItemsControl` и `DataTemplate`. Элементы равномерно распределены через `UniformGrid`.
* **Полная асинхронность:** Загрузка изображений реализована через `HttpClient` с использованием паттерна `async/await`. Интерфейс остается абсолютно отзывчивым (плавным) даже при загрузке изображений очень большого размера.
* **Независимое управление потоками:** Каждая карточка скачивает изображение независимо. Реализован механизм мгновенной отмены загрузки (`CancellationTokenSource`) по кнопке "Stop".
* **Оптимизация памяти:** Используется метод `BitmapImage.Freeze()` для безопасной и эффективной передачи скачанных данных из фонового потока в UI-поток. Экземпляр `HttpClient` инициализируется единожды для предотвращения утечек сокетов.
* **Слабая связность (Loose Coupling):** Логика приложения строго разбита на слои (Models/Services, ViewModels, Views). `MainViewModel` взаимодействует с `ImageItemViewModel` через подписку на события, не нарушая инкапсуляцию.

## Функционал (согласно требованиям)
- [x] 3 независимых слота для отображения загруженных картинок.
- [x] 3 поля ввода URL-адресов.
- [x] Индивидуальные кнопки Start / Stop для каждой картинки.
- [x] Общая кнопка "Load All" для старта всех неактивных загрузок одновременно.
- [x] Динамический общий Progress Bar, отображающий текущее количество активных загрузок (от 0 до 3).

## Как запустить
1. Склонируйте репозиторий: git clone https://github.com/irving2019/PurePalette.git
2. Откройте решение в Visual Studio 2022.
3. Скомпилируйте и запустите проект (F5). Внешние зависимости и NuGet-пакеты не требуются.
